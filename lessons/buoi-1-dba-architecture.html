DBA/GoldenGate — Buổi 1

Architecture Basic: source -> hub -> target

Oracle source -> GoldenGate hub -> PostgreSQL target

1. Bối cảnh thực tế

Một công ty đang dùng Oracle database làm hệ thống chính. Team DBA muốn replicate dữ liệu sang PostgreSQL database để phục vụ báo cáo. Thay vì để Oracle gửi trực tiếp sang PostgreSQL, team đặt GoldenGate trên một máy trung gian gọi là hub.

Trong bài này, bạn chỉ cần giữ một hình trong đầu: Oracle là nơi dữ liệu bắt đầu, GoldenGate hub xử lý ở giữa, PostgreSQL là nơi nhận dữ liệu.

3. Reading: GoldenGate Hub Architecture

A DBA team designs a simple GoldenGate architecture for data replication. The Oracle database is the source, and the PostgreSQL database is the target. Between them, the team places a GoldenGate hub. This hub-based design keeps the replication flow easy to manage.

The topology is simple: Oracle sends changes to the hub, and the hub sends data to PostgreSQL. During the deployment, the DBA checks each service on the hub. One component reads data from the source. Another component sends data to the target. Each database is an endpoint in the system.

If a problem happens, the DBA can explain it clearly: “The source is working, but the hub service has an issue.” This helps the team find the problem faster and keep the target database updated.
